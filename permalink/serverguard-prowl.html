Serverguard-Benachrichtigungen mit Prowl

Seit heute bietet der Serverüberwacher meines Vertrauens, Serverguard24, zusätzlich zu Email und SMS auch Benachrichtigungen via HTTP-Push an. Die Weiterverarbeitung von Benachrichtigungen wird damit deutlich vereinfacht, man könnte zum Beispiel auf leicht zu lösende Probleme mit einem automatischen Neustart von Diensten a la Monit reagieren. Oder man kann sich seine Benachrichtigungen so unbauen, dass sie via Prowl direkt auf das iPhone gepusht werden.

Schon weil ich bei Prowl über die Wichtigkeit einer Nachricht definieren kann, ob sie mich auch nachts aus dem Schlaf reißen darf, ist das ein deutlicher Mehrwert gegenüber SMS.

Bei HTTP-Push übergibt Serverguard alle Details einer Benachrichtigung per POST an eine beliebige Url. Da wiederrum Prowl über die API per einfachem GET-Request anzusteuern ist, ist ein Script zum weiterleiten der Nachrichten schnell gebaut - es müssen ledigliche die Werte von Serverguard in die zwei Felder "event" und "description" von Prowl nach Wunsch umfortmatiert werden.
Als kleines Gimmick habe ich noch eine dynamische Priorisierung eingebaut, damit anhand der Nachrichten-Kategorie (OK, WARNING, CRITICAL) eine Prowl-Prio zwischen Normal und Emergency gesetzt wird.
Das Skript findet sich bei Github als serverguard-prowl-notifier und muss nur noch um den eigenen API-Key erweitert und auf den Webspace der Wahl hochgeladen werden.
