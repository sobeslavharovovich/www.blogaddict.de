iPhone-Push für FTP-Uploads

Wenn man einen FTP-Server hauptsächlich dazu betreibt, dort vereinzelte Dateien von anderen Leuten zu empfangen, kann es lohnen über Uploads informiert zu werden. Pure-FTPd bietet - abseits von einer sehr einfachen Einrichtung - die schöne Möglichkeit für jeden abgeschlossenen Download ein Skript auszuführen.


Basierend auf meinem Prowl-Skript für Serverguard ist schnell eine Upload-Benachrichtigung für Pure-FTP gebastelt: pureftp_uploadscript_prowl.php


Das Skript nimmt den neuen Dateinamen aus der Kommandozeile entgegen und verarbeitet die per Environment übergebenen Informationen zu Dateigröße und Besitzer in eine Prowl-Nachricht. Zur Installation muss das Skript nur in /usr/local/bin kopiert, mit chmod +x pureftp_uploadscript_prowl.php ausführbar gemacht und im Skript selbst die Variable $apikey mit dem Prowl API-Key gefült werden. Die Integration in pureftpd geschieht bei einem Debian-System ganz einfach über drei Einträge in /etc/defaults/pure-ftpd-common:


UPLOADSCRIPT=/usr/local/bin/pureftp_uploadscript_prowl.php

UPLOADUID=<uid des ftpusers>

UPLOADGID=<gid des ftpusers>


Bei anderen Distributionen muss der Aufruf von pure-uploadscript wie folgt angepasst werden:


pure-uploadscript -r /usr/local/bin/pureftp_uploadscript_prowl.php -B -u <uid des ftpusers> -g <gid des ftpusers>


Schließlich noch ein Neustart von pure-ftpd und künftig wird jeder erfolgreiche Upload mit einer Push-Notification auf dem iPhone angekündigt!
