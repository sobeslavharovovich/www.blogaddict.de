iPhone-Push für FTP-Uploads

Wenn man einen FTP-Server hauptsächlich dazu betreibt, dort vereinzelte Dateien von anderen Leuten zu empfangen, kann es lohnen über Uploads informiert zu werden. Pure-FTPd bietet - abseits von einer sehr einfachen Einrichtung - die schöne Möglichkeit für jeden abgeschlossenen Download ein Skript auszuführen.


Basierend auf meinem Prowl-Skript für Serverguard ist schnell eine Upload-Benachrichtigung für Pure-FTP gebastelt: pureftp_uploadscript_prowl.php


Das Skript nimmt den neuen Dateinamen aus der Kommandozeile entgegen und verarbeitet die per Environment übergebenen Informationen zu Dateigröße und Besitzer in eine Prowl-Nachricht. Zur Installation muss das Skript nur in /usr/local/bin kopiert, mit chmod +x pureftp_uploadscript_prowl.php ausführbar gemacht und im Skript selbst die Variable $apikey mit dem Prowl API-Key gefült werden. Die Integration in pureftpd geschieht bei einem Debian-System ganz einfach über drei Einträge in /etc/defaults/pure-ftpd-common:


UPLOADSCRIPT=/usr/local/bin/pureftp_uploadscript_prowl.php

UPLOADUID=<uid des ftpusers>

UPLOADGID=<gid des ftpusers>


Bei anderen Distributionen muss der Aufruf von pure-uploadscript wie folgt angepasst werden:


pure-uploadscript -r /usr/local/bin/pureftp_uploadscript_prowl.php -B -u <uid des ftpusers> -g <gid des ftpusers>


Schließlich noch ein Neustart von pure-ftpd und künftig wird jeder erfolgreiche Upload mit einer Push-Notification auf dem iPhone angekündigt!

gFTP Passwort sichtbar machen

Möglicherweise ist das ein Einzelschicksal, aber mir passiert es regelmäßig, dass ich ein FTP-Passwort zwar in gFTP gespeichert habe, aber nirgendwo mehr im Klartext nachsehen kann. Auf dem Mac ist das dank Schlüsselbundverwaltung kein Thema, GNOME bietet aber nichts vergleichbares bzw. gFTP nutzt es nicht.


Nun speichert gFTP die Passwörter aber nicht verschlüsselt, sondern macht sie nur schwerer lesbar. Das ist zwar für die Sicherheit insgesamt etwas unglücklich, ermöglicht aber das auslesen des Passworts. George Notaras hat den Code des Descramblers von C in Python übersetzt und durch einfaches anhängen von zwei Zeilen zur Ausgabe habe ich nun ein Tool um Passwörter sichtbar zu machen.


Alles was nötig ist, ist das zerhackte Passwort aus der gFTP-Config zu holen und dann den Descrambler auf der Kommandozeile aufzurufen:

python descramble_gftp.py $scrambledPass$


Den Code für den fertigen Descrambler (der wie gesagt ganze zwei Zeilen Eigenleistung von mir enthält, aller Dank gilt hier also George Notaras) hab ich bei Github abgelegt: descramble_gftp

Twhirl läuft unter Linux

Offiziell funktioniert der Twitter-Client Twhirl nur unter Windows und Mac OSX - eben weil Adobe Air nur dort verfügbar ist und Twhirl darauf läuft.


Adobe arbeitet aber aktuell an einer Linux-Umsetzung von Air und hat bereits eine öffentliche Alpha-Version zum Download gestellt. Darüber lässt sich Twhirl erstaunlich problemlos auf Linux betreiben.


So gehts:

Als erstes muss Adobe Air Alpha für Linux heruntergeladen und installiert werden. Die Adobe Labs bieten den Download ohne Registrierung oder ähnlich Hürden an. (Direkter Download: Adobe Air Linux A1 033108).

Die heruntergeladene Datei muss per chmod +x adobeair_linux_a1_033108.bin ausführbar und dann mit ./adobeair_linux_a1_033108.bin gestartet werden. Der Installer sollte selbsterklärend sein. Danach noch Firefox neu starten und dann auf der Twhirl-Seite den Direktdownload des aktuellen Releases starten. Der automatische Installer auf der Seite geht nicht, etwas tiefer gibt es aber Instruktionen für den manuellen Download (direkter Download von Twhirl 0.8.2). Firefox sollte nur automatisch vorschlagen, dass die .air-Datei mit dem Adobe Air Installer geöffnet wird. Nach der Installation ist Twhirl auf Gnome im Menü "Sonstiges" zu finden und sieht bis auf die fehlende Transparenz am Fensterrahmen wie gewohnt aus.
